4. ServiceNow Business Rules (Server-Side Automation)

RuleTableTriggerAction
BR-1 event_registrations After INSERT Increments registered_count on the parent event record
BR-2 event_registrations Before DELETE Decrements registered_count on the parent event record
BR-3 sys_user After UPDATE when approval_status → approved Creates an approval notification for the volunteer
BR-4 volunteer_assignments After INSERT Creates an assignment notification for the assigned volunteer
BR-5 event_registrations After INSERT Creates an event registration notification for the volunteer

5. Login & Role Separation

🔑 Admin Role

Logs in via the Admin tab. If a volunteer account tries the Admin tab — blocked with an error message.

Can access:

Dashboard · Volunteers · Projects · Assignments · Events · Impact Reports · Notifications

👤 Volunteer Role

Logs in via the Volunteer tab. Must be active=true and approval_status=approved in ServiceNow.

Can access:

Volunteer Dashboard · Notifications

New Volunteer Sign-Up Flow

1. Volunteer signs up on login page → sys_user record created: active=false, approval_status=pending 2. Admin sees them in "Pending Approval" tab → clicks Approve → active=true, approval_status=approved → BR-3 fires → notification created: "Your registration has been approved" 3. Volunteer can now log in and access the Volunteer Dashboard

8. Key Data Flows

Admin Creates an Assignment

Admin clicks "Create Assignment" → POST /x_2048396_ngo_vo_1_volunteer_assignments → BR-4 fires in ServiceNow → notification record created for the volunteer (type = assignment) → Volunteer sees: "You have been assigned to project: [name]"

Volunteer Registers for an Event

Volunteer clicks "Register" → POST /x_2048396_ngo_vo_1_event_registrations → BR-1 fires → event.registered_count + 1 → BR-5 fires → notification created (type = event) → Spots-left counter decrements on next page load

Volunteer Unregisters from an Event

DELETE /x_2048396_ngo_vo_1_event_registrations/{id} → BR-2 fires → event.registered_count - 1 → Spot becomes available again for other volunteers

Admin Approves a Volunteer

Admin clicks "Approve" on Pending tab → PATCH /sys_user/{id} { active: true, approval_status: approved } → BR-3 fires → notification created: "Your volunteer registration has been approved. Welcome to the team!" → Volunteer can now log in with their credentials
